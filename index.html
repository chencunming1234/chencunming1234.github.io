初识github

进入github官网github.com
创建github账号

1.添加repositories

2.设置ssh-keygen;
命令：vim ~/.ssh/id_rsa.pub

把得到的ssh-key复制出来
:q(退出复制界面)代码规范如下图

添加到github的对应位置

3.git clone+ssh地址，就可以把我们的repositories下载到本地
git status 查看当前的仓库的状态

4.修改更新：git config –global user.name “Your Name”
git config –global user.email “email@example.com”
（首次上传时需要配置本地信息）
5.上传到暂存区
git add .
git commit -m “日志”
6.上传到github:
git remote add origin git@github.com:yourname/learngit.git
git push -u origin master

##为github绑定一个hexo博客

1.安装hexo
hexo install -g
2.初始化hexo
hero init
3.添加第三方模块
npm install hexo-deployer-git –save
4.把github创建的repositories的http://地址写入到_config.yml文件尾部的deploy中；
deploy:git;
repo:https://github.com/chencunming1234/chencunming1234.github.io.git（your adress）
5.返回终端运行
hexo g
hexo d

初识npm

npm的安装

npm install -g;

-g表示全局安装

npm adduser 创建账号

第三方模块：

npm install+模块名称（下载第三方模块到本地）

自定义模块发行到npm

npm init （初始化）
（含有：package.son;index.js;README.md）
npm publish （发行上传）

npm unpublish+模块名称@版本号（删除发行）

sudo npm -g hexo （全局下下载安装hero）

初识node.js

下载安装node.js;

进入官网：https://nodejs.org/en/

做开发的时候最好还是用LTS稳定版的，平时学习的时候可随意！

js与node.js的一些区别


1、b/s系统；
浏览器/服务端

chrome浏览器//两个内核 webkit v8

webkit:页面布局，解析样式
v8:JS解析器
硬件层：

node.js的底层基于c

libuv > libev >连接c/c++ > 硬件层

客户端JS（ECMA/DOM/BOM）
核心为ECMA
DOM/BOM是浏览器自己封装的

在node.js里面只有ECMA，没有BOM DOM window

znode.js的API:process os file buffer util queryting
node.js特色：没有兼容性问题;可运行ecma6;


### node.js的速度远比js快的多

##node.js的安装

到官网下载好安装包以后直接点击运行一直到安装结束


在终端里可检查是否有安装好；下面以mac系统为例



node -v；出现node.js的版本号就说明已经安装完成
