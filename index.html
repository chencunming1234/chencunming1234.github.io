对象属性及原型属性优先级等。

这个题目先创建了一个叫Foo的函数，然后为他定义了一个getName的方法；拉着就为Foo的原型添加了一个getName的方法；
再通过函数变量表达式创建了个getName的匿名函数；最后声明了一个具有函数名getName的函数；

函数的解析过程:http://www.2cto.com/kf/201401/273825.html

下面先公布了下答案：

1、这个题目只是执行了Foo上的自己的方法。
2、这个题目要注意两个地方:
（1）javascript在解析的时候会把变量名提取出来，不包含变量的值，等要用的时候再去查找相应的值
（2）javascript在解析的时候会把整个函数都提取出来，包含整个函数的内容，如果函数名与变量相同就保留函数名；
所以这个题目执行时候是这样的，var getName被提取出来，function getName() {
alert (5);}整个内容都被提取出来；然后顺序执行，执行过程中把function () {
alert (4);};重新赋值给getName;所以答案为4。
3、这个题目先执行了Foo函数，同时由于里面的getName不带var，是一个全局变量，改变了getName的内容；然后返回了this;这时候

this指向的是window;相当于window.getName();但是getName的值已经被function () {
alert (1); };覆盖；所以答案为1。
4、由于在上一题中getName的内容已经被改变；所以得到的答案还是1。
5、这个题目涉及到了运算符的优先级问题，相当优先级的内容自行百度。这里“.”的优先级大于new;所以相当于

所以答案为3。
7、根据优先级来算，相当于new ((new Foo()).getName) ();
先执行new Foo(),再执行.getName,最后才执行第一个new;
所以答案为3。

伪类与伪元素

伪类

伪类一般都带有一种状态；例如：a:link;a:visited;a:hover;a:active;他们分别表达了一种状态

连接、访问、划过、激活；

伪元素

首先，伪元素是没有语义的，它不写在HTML里。下面列出一些常见的伪元素：
::before;它表示在尾部追加一个标签，
::after;它表示在标签或内容前面增加一个标签
::first-line;它表示选中文本的第一行；
::first-letter;它表示选中文本的首字符；
::selection;它表示鼠标选中的内容（只能更改两个属性值：background和color）

对于before和after：它们出现在标签或文本的前面或者后面；在有标签时出面在标签前面或者后面，有广西
时出现在文本内容的开始和结束的位置！

before和after可配合position使用，可以适合的时候在标签前面增加效果！
div1::before{
content: “”;
width: 50px;
height: 50px;
background: black;
border-radius: 50%;
display: block;position: absolute;
left: -10px;top:-10x;
}
这样可以对象标签的适当位置增加一个效果

.div1::after{
contne:””;
clear:both;
display:block;
height:0;
zoom:1;
}
这们一段代码具有清除浮动的效果；display/height/zoom是为了兼容低版本的浏览器；

初识github

进入github官网github.com
创建github账号

1.添加repositories

2.设置ssh-keygen;
命令：vim ~/.ssh/id_rsa.pub

把得到的ssh-key复制出来
:q(退出复制界面)代码规范如下图

添加到github的对应位置

3.git clone+ssh地址，就可以把我们的repositories下载到本地
git status 查看当前的仓库的状态

4.修改更新：git config –global user.name “Your Name”
git config –global user.email “email@example.com”
（首次上传时需要配置本地信息）
5.上传到暂存区
git add .
git commit -m “日志”
6.上传到github:
git remote add origin git@github.com:yourname/learngit.git
git push -u origin master

deploy:git;
repo:https://github.com/chencunming1234/chencunming1234.github.io.git
5.返回终端运行

hexo d

hexo g == hexo generate
